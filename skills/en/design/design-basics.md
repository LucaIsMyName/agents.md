# design-basics.md

## Purpose

This document defines **what good design is and how to create it** across:

- product design
- UI/UX
- visual design
- systems and services

The goal: design that is **useful, understandable, and built to last**.

---

## Core Mindset

Good design is:

- **user-centered**
- **simple**
- **intentional**

👉 Design is not decoration. It is how something **works**.

---

## 1. Start With the Problem

### Rule

Understand the **real need** before designing.

### ❌ Bad

Designing based on assumptions

### ✅ Good

- identify the user
- understand their goal
- define the problem clearly

👉 If the problem is wrong, the design will be wrong.

---

## 2. Make It Useful

### Rule

Design must solve a real problem.

👉 Remove anything that does not add value.

---

## 3. Keep It Simple

### Rule

Reduce to the essential.

### ❌ Bad

Too many features, options, or visuals

### ✅ Good

- fewer elements
- clear structure
- focused functionality

👉 Simplicity requires effort.

---

## 4. Make It Understandable

### Rule

The user should not have to think.

### ❌ Bad

Unclear navigation, hidden actions

### ✅ Good

- obvious actions
- clear labels
- predictable behavior

👉 Good design explains itself.

---

## 5. Be Consistent

### Rule

Use consistent patterns.

👉 Same action = same result everywhere

- layouts
- interactions
- naming

👉 Consistency builds trust.

---

## 6. Prioritize Clarity Over Style

### Rule

Visual design supports function.

### ❌ Bad

Design that looks good but confuses

### ✅ Good

- readable typography
- clear hierarchy
- strong contrast

👉 If it’s hard to use, it’s bad design.

---

## 7. Design for Real Use

### Rule

Consider real-world conditions:

- distractions
- errors
- different devices
- accessibility

👉 Design should work in imperfect situations.

---

## 8. Provide Feedback

### Rule

Every action should have a response.

### Examples

- loading indicators
- success/error messages
- state changes

👉 Users need to know what is happening.

---

## 9. Prevent Errors

### Rule

Design should **reduce mistakes**.

### ❌ Bad

Easy to misuse interface

### ✅ Good

- constraints
- confirmations
- clear affordances

👉 Prevention is better than correction.

---

## 10. Be Honest

### Rule

Do not mislead users.

### ❌ Bad

Dark patterns, fake urgency

### ✅ Good

- transparent actions
- clear consequences

👉 Trust is part of good design.

---

## 11. Design for Accessibility

### Rule

Design must work for as many people as possible.

- readable text
- sufficient contrast
- keyboard/navigation support

👉 Accessibility is not optional.

---

## 12. Focus on Hierarchy

### Rule

Guide attention.

### Use:

- size
- spacing
- contrast
- positioning

👉 Users should instantly see what matters.

---

## 13. Iterate and Improve

### Rule

Design is never finished.

- test with users
- observe behavior
- refine continuously

👉 Good design evolves.

---

## 14. Remove the Unnecessary

### Rule

Every element must justify its existence.

👉 If in doubt, remove it.

---

## 15. Common Footguns

### 🚨 Overdesign

→ too complex, too many features

### 🚨 Designing for yourself

→ not for real users

### 🚨 Prioritizing aesthetics over usability

→ looks good, works poorly

### 🚨 Inconsistency

→ confusing experience

### 🚨 Ignoring edge cases

→ breaks in real use

## Final Thought

Good design feels:

- obvious
- calm
- effortless

Bad design feels:

- confusing
- noisy
- frustrating

👉 If users have to think too much, the design is not good.
